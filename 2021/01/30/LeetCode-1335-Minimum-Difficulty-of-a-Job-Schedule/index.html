LeetCode 1335 Minimum Difficulty of a Job Schedule
Publish: 2021/1/30   
  • dp

Description:

​ Features: returning the minimum value instead of finding the specific job schedule, so it is typical of dynamic programming.

​ The key to dynamic programming is, first, to find the “State Transition Equation”, and second, to determine the “initial values”.

​ So at first, let’s find out the so-called “State Transition Equation”. When we solve algorithmic problems, we generally start with intuition, e.g., brutal force is the intuition. In this dynamic programming problem, we are asked to plan our jobs, so we try to see from our “intuition” if we can make a schedule.

​ The question requires us to do at least one job a day, so each time we arrive at a new day, we can choose to do one job, or two jobs, or three jobs, and so on. Then by the next day, we can still choose to do one job, or two jobs, or three jobs.

​ So how should we choose? The question asks us to return the minimum difficulty of a job schedule, each day has multiple choices, and all the days constitute multiple combinations, that is, we have to choose minimum difficulty among all the combinations. Well, this is our “intuition”.

​ Next, let’s use a diagram to show the specific process, so that you can understand the specific process of ”State Transition”. Suppose we do 5 jobs in 3 days, with the X-axis representing time and the Y-axis representing jobs.

​ Then the question arises, what does each grid mean? It means: the minimum difficulty to complete j jobs in d days. The first grid means “the minimum difficulty of completing the first job on the first day”, obviously the minimum difficulty of completing a job on the first day can only be the difficulty of the first job.

​ Let’s look down the grid, this cell represents “the minimum difficulty of completing two jobs on the first day”, obviously, it should be the most difficult of the two jobs, because the question specifies that the difficulty of each day is the most difficult of all the jobs done that day.

​ Hmm. The next grid does three jobs on day 1, then. At the bottom is doing all 5 jobs on day 1. Okay, next we’ll look at column 2, there’s only one job to do, which is impossible because at least one job is done every day, so it’s impossible to do one job for two days, so we set this value to be infinite, and then the next grid is two days to do 2 jobs, so that’s only one choice, which is to do the 1st job on day 1 and the 2nd day to do the second job. Then in the next grid inside is two days to do three jobs, at this time, we have two choices, is to do the first job on the first day, the second day to do 2,3 jobs, or to do the first 2 job on the first day then do the third job on second day.

​ So the final value is:

min(do k jobs on (d-1)th day + do j-k jobs on d-th day)

​ Here is the code:

class Solution:
    def minDifficulty(self, jobDifficulty: List[int], d: int) -> int:
      jobLen = len(jobDifficulty)
      dp = [[sys.maxsize] * d for _ in range(jobLen)]
      dp[0][0] = jobDifficulty[0]
      for i in range(1, jobLen):
          dp[i][0] = max(dp[i - 1][0], jobDifficulty[i])
      for dayIdx in range(1, d):
          for jobIdx in range(1, jobLen):
              for jobDoneIdx in range(jobIdx):  # jobDoneIdx: how many jobs have been done before this day 
                  dp[jobIdx][dayIdx] = min(dp[jobIdx][dayIdx],
                                           dp[jobDoneIdx][dayIdx - 1] + max(jobDifficulty[jobDoneIdx + 1:jobIdx + 1]))
      return dp[-1][-1] if dp[-1][-1] != sys.maxsize else -1

​ Okay, the time complexity of this solution may be a little bit slow because inside the three-level loop, we need to get the maximum value among the remaining jobs to do on the last day. This will take a lot of extra time. But it seems we have no other choice, because we start with job 1, then job 2, then job 3, and if we do them sequentially, we don’t know which job is the most difficult in the future, so we have to traverse all the rest of jobs. So in this case, how can we know which of the future jobs is the most difficult? We can consider another way, that is, Let’s imagine that we have a time machine, and we drive it to the last job in the future. Then we loop backwards, “just keep the only maximum value each time and keep iterating backwards”, so that we can get the maximum value without iterating through all the rest of jobs each time. So now let’s look at the code.

class Solution:
    def minDifficulty(self, jobDifficulty: List[int], d: int) -> int:
        jobLen = len(jobDifficulty)
        maxVal = int(sys.maxsize/2)
        dp = [[maxVal] * d for _ in range(jobLen)]
        dp[0][0] = jobDifficulty[0]
        for i in range(1, jobLen):
            dp[i][0] = max(dp[i - 1][0], jobDifficulty[i])
        for dayIdx in range(1, d):
            for jobIdx in range(1, jobLen):
                maxDiff = 0
                for jobDoneIdx in range(jobIdx, 0, -1):
                    maxDiff = max(maxDiff, jobDifficulty[jobDoneIdx])
                    dp[jobIdx][dayIdx] = min(dp[jobIdx][dayIdx], dp[jobDoneIdx - 1][dayIdx - 1] + maxDiff)
        return -1 if dp[-1][-1] == maxVal else dp[-1][-1]
