Leetcode 1192 Critical Connections in a Network
Publish: 2021/6/7   
  • graph

Let us skip the description. Try to understand how can we tell the “Critical Connection”.

​ Right, the definition is:

​ Any connection IN a circle is NOT the “Critical Connections”.

​ Any connection NOT IN a circle is the “Critical Connections”.

​ How can our naked eyes tell what is the so-called “Critical Connection” from the example? Yes, our eyes just consider all the nodes in a same circle as “one single node”, and the left connections will be the result we need.

​ And how can we define a circle? OK, we use DFS to go through all the nodes and connections, we give them IDs, if the node are in the same circle, they will be given a same ID. After the “give ID” within “DFS”, we could tell the “critical connections”.

class Solution:
    def criticalConnections(self, n: int, connections: List[List[int]]) -> List[List[int]]:
        map = collections.defaultdict(set)
        for pair in connections:
            map[pair[0]].add(pair[1])
            map[pair[1]].add(pair[0])
        ids = [-1] * n
        ans = []
        self.dfs(0, 20, -1, ids, map, ans)
        return ans

    def dfs(self, node: int, nodeId: int, parent: int, ids: [int], map, ans) -> int:
        ids[node] = nodeId
        subSet = map[node]
        for neighbor in subSet:
            if neighbor == parent:  # parent, do not visit
                continue
            elif ids[neighbor] == -1:  # not visit yet, go deep
                ids[node] = min(ids[node], self.dfs(neighbor, nodeId+1, node, ids, map, ans))
            else:
                ids[node] = min(ids[node], ids[neighbor])  # already visited, update min
        if ids[node] == nodeId and node != 0:
            ans.append([parent, node])
        return ids[node]
